# Accessibility Improvements Design — #115

Four a11y sub-features: aria-live announcements, form button labels, keyboard tree navigation, keyboard drag-drop.

## 1. aria-live / announcements

- Add `role="alert"` to cut notification in `App.tsx` (the only banner missing it)
- Add `aria-label` to AutosaveBanner buttons (Restore, Dismiss) — currently have `title` only

## 2. Form button labels

- Add `aria-label` to DetailSidebar action buttons: Save ("Save changes"), Delete ("Delete person"), Edit ("Edit person"), Clear selection ("Clear selection")
- Form input labels are already present on all fields — no changes needed

## 3. Keyboard tree navigation

Vim mode (`useVimNav.ts`) already provides `j/k/h/l` spatial navigation. Add arrow key aliases in the same key handler:
- `ArrowDown` = `j` (next node down)
- `ArrowUp` = `k` (next node up)
- `ArrowRight` = `l` (expand / child)
- `ArrowLeft` = `h` (collapse / parent)

Add `Shift+Tab` support in PersonNode edit mode for reverse field cycling (currently only forward Tab works).

## 4. Keyboard drag-drop + selection

- Add `KeyboardSensor` from `@dnd-kit/core` alongside existing `MouseSensor` in `useChartLayout.ts`. dnd-kit handles Space to pick up, arrow keys to move, Space to drop, Escape to cancel.
- Add `Ctrl+A` / `Cmd+A` select-all handler. Calls `batchSelect` with all visible person IDs.
- Lasso stays mouse-only — Ctrl+A and Shift+Click provide keyboard alternatives.
